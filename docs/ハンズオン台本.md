# ハンズオン台本

Docker を `app`、`db`、`mail` の順に少しずつ追加して理解するための軽い台本。

## Step 1: Dockerfile だけで app を起動する

まずは Compose を使わず、Dockerfile だけで PHP + Apache のコンテナを起動する。

見るファイル:

```text
step1_app/docker/app/Dockerfile
step1_app/src/index.php
```

話すポイント:

- Dockerfile は「イメージの作り方」を書くファイル。
- `FROM php:8.3-apache` で PHP と Apache が入った公式イメージを土台にする。
- `WORKDIR /var/www/html` はコンテナ内の作業ディレクトリを指定している。
- この段階では `src/index.php` を Dockerfile で `COPY` しない。
- 代わりに `docker run -v` でホストの `src` をコンテナへマウントする。
- `docker run -e` でコンテナに環境変数を渡せる。
- PHP 側では `getenv()` で環境変数を読める。

実行:

```bash
cd step1_app
docker build -t handson-step1-app -f docker/app/Dockerfile .
docker run --rm --name handson-step1-app -e APP_MESSAGE="Hello from docker run environment" -e APP_ENV=handson -p 8080:80 -v "$PWD/src:/var/www/html" handson-step1-app
```

確認:

```text
http://localhost:8080
```

コンテナ内の環境変数を確認する:

```bash
docker exec handson-step1-app printenv APP_MESSAGE
docker exec handson-step1-app printenv APP_ENV
```

PHP から環境変数を読めるか確認する:

```bash
docker exec handson-step1-app php -r 'echo getenv("APP_MESSAGE") . "\n" . getenv("APP_ENV") . "\n";'
```

確認できたら、別ターミナルで停止する。

```bash
docker stop handson-step1-app
```

ここで理解したいこと:

- `docker build` はイメージを作る。
- `docker run` はイメージからコンテナを起動する。
- `-e APP_MESSAGE=...` はコンテナ内の環境変数を指定する。
- `-p 8080:80` はホストとコンテナのポートをつなぐ。
- `-v "$PWD/src:/var/www/html"` はホストのファイルをコンテナに見せる。

## Step 2: Compose で app と db を起動する

次に Compose を使って、`app` と `db` をまとめて起動する。

見るファイル:

```text
step2_app_db/docker-compose.yml
step2_app_db/docker/app/Dockerfile
step2_app_db/docker/db/init/001_create_tables.sql
step2_app_db/src/index.php
step2_app_db/.env.example
```

話すポイント:

- Compose は複数コンテナの起動設定を `docker-compose.yml` にまとめるためのもの。
- `app` は Dockerfile からビルドする。
- `db` は MySQL 公式イメージをそのまま使う。
- `app` から MySQL へは `localhost` ではなく `db` というサービス名で接続する。
- `db_data:/var/lib/mysql` で DB のデータを volume に保存する。
- `docker/db/init` 配下の SQL は、DB volume 初回作成時に実行される。

実行:

```bash
cd step2_app_db
cp .env.example .env
docker compose config
docker compose up -d --build
```

確認:

```text
http://localhost:8080
```

画面に次のような表示が出れば成功。

```text
DB status: Step 2: app connected to db
```

状態確認:

```bash
docker compose ps
```

停止:

```bash
docker compose down
```

初期 SQL からやり直す場合:

```bash
docker compose down -v
```

ここで理解したいこと:

- Compose を使うと `build`、`ports`、`volumes`、`environment` をファイルに残せる。
- 同じ Compose ネットワーク内ではサービス名で通信できる。
- DB のデータはコンテナではなく volume に残す。
- `depends_on` は起動順の指定であり、DB の完全な準備完了までは保証しない。

## Step 3: mail を追加する

最後に Mailpit を追加して、`app` から `mail` に接続できることを確認する。

見るファイル:

```text
step3_app_db_mail/docker-compose.yml
step3_app_db_mail/docker/app/Dockerfile
step3_app_db_mail/src/index.php
step3_app_db_mail/.env.example
```

話すポイント:

- `mail` は Mailpit 公式イメージを使う。
- アプリがメールを送る先は `mail:1025`。
- 人間がブラウザで確認する画面は `localhost:8025`。
- `1025` は SMTP 受信用、`8025` は Web UI 用。
- `app`、`db`、`mail` を同じ `app_network` に入れることで、サービス名で通信できる。

実行:

```bash
cd step3_app_db_mail
cp .env.example .env
docker compose config
docker compose up -d --build
```

App を確認:

```text
http://localhost:8080
```

Mailpit を確認:

```text
http://localhost:8025
```

画面に次のような表示が出れば、`app` から `mail` に接続できている。

```text
Mail status: mail:1025 is reachable
```

状態確認:

```bash
docker compose ps
```

停止:

```bash
docker compose down
```

ここで理解したいこと:

- 外から見るポートと、コンテナ間で使うポートは別に考える。
- ブラウザで見るのは `localhost:8025`。
- コンテナから接続するのは `mail:1025`。
- Compose はサービスが増えても、設定を1ファイルにまとめられる。

## 進行メモ

各 step を始める前に、前の step を止めておく。

```bash
docker compose down
```

Step 1 の場合は Compose ではないので、起動中なら次で止める。

```bash
docker stop handson-step1-app
```

同じポートを使うため、複数 step を同時に起動しない。

Step 2 以降で作る `.env` はローカル環境用なので Git には含めない。公開するのは `.env.example`。
