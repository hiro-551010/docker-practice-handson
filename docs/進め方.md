# 進め方

このハンズオンでは、最初から完成形を作らない。

1つずつコンテナを追加し、そのたびに Compose の設定、起動状態、ログ、ブラウザ表示を確認する。

## Step 1: app だけ起動

使うフォルダ:

```bash
cd step1_app
```

目的:

- 自作 Dockerfile を作る
- `docker build` でイメージを作る
- `docker run` でコンテナを起動する
- `docker run -e` で環境変数を渡す
- ホストの `src` をコンテナに bind mount する
- `localhost:8080` で PHP を表示する

この段階では DB も mail も使わない。

## Step 2: db を追加

使うフォルダ:

```bash
cd step2_app_db
```

目的:

- MySQL 公式イメージを使う
- `.env` から DB 情報を渡す
- `db_data` volume で MySQL のデータを永続化する
- `app` から `db:3306` で接続する

## Step 3: mail を追加

使うフォルダ:

```bash
cd step3_app_db_mail
```

目的:

- Mailpit 公式イメージを使う
- `mail:1025` に SMTP 接続する
- `localhost:8025` で受信メールを確認する

## 毎ステップで確認すること

Step 1 は Dockerfile 単体で進める。

```bash
docker build -t handson-step1-app -f docker/app/Dockerfile .
docker run --rm --name handson-step1-app -e APP_MESSAGE="Hello from docker run environment" -e APP_ENV=handson -p 8080:80 -v "$PWD/src:/var/www/html" handson-step1-app
```

Step 2 以降は Compose を使う。

`.env.example` からローカル用の `.env` を作る。

```bash
cp .env.example .env
```

Compose の解決結果を確認する。

```bash
docker compose config
```

起動する。

```bash
docker compose up -d --build
```

状態を見る。

```bash
docker compose ps
```

ログを見る。

```bash
docker compose logs -f
```

停止する。

```bash
docker compose down
```

Step 1 は Compose ではないため、停止は次のコマンドで行う。

```bash
docker stop handson-step1-app
```

`docker run` に `--rm` を付けず `-d` で裏起動した場合は、停止後に `docker rm handson-step1-app` でコンテナも削除する。

## 並走モード (3 step を同時に動かす)

ハンズオンで進めるときは前の step を止めてから次に進む。すべての step を裏で動かし続けたい場合は、`.env` でホストポートを step ごとにずらして競合を避ける。

| step | APP_PORT | DB_PORT | MAIL_SMTP_PORT | MAIL_WEB_PORT |
| --- | --- | --- | --- | --- |
| step1 | 18080 | - | - | - |
| step2 | 28080 | 23306 | - | - |
| step3 | 38080 | 33306 | 31025 | 38025 |

step1 は Compose ではないので、並走させる場合は `--rm` を外して `-d` で起動する。

```bash
cd step1_app
docker run -d --name handson-step1-app \
  -e APP_MESSAGE="Hello from docker run environment" -e APP_ENV=handson \
  -p 18080:80 -v "$PWD/src:/var/www/html" handson-step1-app
```

step2 / step3 は各フォルダで `.env` のポートを書き換えてから `docker compose up -d` する。コンテナ側のポート (`80` / `3306` / `1025` / `8025`) は変えない。
